# Protocol Descriptors

(Phase C3 reference document.)

## Concept

Each protocol is described by a declarative **descriptor** — a single file under
`lib/protocols/<name>.uc` that registers itself with the central registry on
load. The descriptor carries:

- `kind`: `"inbound"` or `"outbound"`
- `type`: UCI type tag (e.g. `"ssh"`)
- `sing_box_type`: the value sing-box expects in the JSON `"type"` field
- `fields[]`: declarative field list with type/validate/required/secret hints
- `emit(section)`: function that takes a UCI section and returns the sing-box JSON

## How dispatch works

`lib/outbound.uc::build_constructor_for(s, t)` (and `lib/inbound.uc` analog)
consults the registry FIRST. If a descriptor is registered for `(outbound, t)`,
its `emit()` is called and the result returned. Otherwise the legacy
switch-by-type logic runs.

This allows incremental migration. New protocols (like the SSH outbound shipped
in C3) live as descriptors; legacy protocols stay in the switch until migrated.

`lib/outbound.uc` eagerly requires each descriptor module at load time so
its `register()` call fires. The require is wrapped in a try/catch — a
missing or broken descriptor never breaks the legacy dispatcher.

## Writing a descriptor

```ucode
// lib/protocols/my_protocol.uc
let reg = require("protocols.registry");
reg.register({
    kind: "outbound",
    type: "my_protocol",
    sing_box_type: "my_protocol",
    fields: [
        { name: "server", type: "string", required: true, validate: "host" },
        { name: "server_port", type: "number", default: 443, validate: "port" },
        { name: "password", type: "string", secret: true },
    ],
    emit: function(s) {
        return {
            type: "my_protocol",
            tag: s[".name"],
            server: s.server,
            server_port: +s.server_port || 443,
            password: s.password,
        };
    },
});

return {};
```

Then add one line to `lib/outbound.uc` near the top:

```ucode
try { require("protocols.my_protocol"); } catch (_) {}
```

…and add the file to `scripts/install-manifest.txt` (regenerated by
`scripts/gen-manifest.sh`).

## Module resolution

`require("protocols.registry")` and `require("protocols.ssh")` use ucode's
dotted module syntax. The runtime's `-L /usr/share/singbox-ui/lib` search
path resolves these to `lib/protocols/registry.uc` and `lib/protocols/ssh.uc`
respectively. No additional `-L` flags are needed at any invocation site.

## Field type vocabulary (current)

- `string` / `number` / `bool`
- `list` with `item: "string"` (UCI list option)

## Field hint vocabulary

- `required: true` — UI will validate non-empty.
- `default: <value>` — emit value if section field is null.
- `secret: true` — UI shows masked input; output goes through `lib/scrub.uc` on RPC paths.
- `validate: "host"|"port"|"path"` — hint for UI validators.

These hints are advisory in C3; full frontend descriptor-driven rendering and
strict backend validation per descriptor are tracked as Phase D work.

## Migration plan

Phase C3 ships the registry + one reference descriptor (SSH). Phase D will
migrate the existing 7 protocols (vless, vmess, trojan, hysteria2, tuic, anytls,
shadowsocks) one at a time, removing the corresponding switch branch from
`lib/outbound.uc` / `lib/inbound.uc` after each.

After each migration:
- `lib/protocols/<name>.uc` ships the descriptor
- Switch branch removed from the legacy dispatcher
- Existing tests for the protocol still pass against the descriptor's emit()
